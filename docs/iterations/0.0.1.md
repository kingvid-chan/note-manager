# 迭代 0.0.1 — 笔记管理系统 MVP

## 范围

完整 MVP：用户注册登录、笔记 CRUD、搜索、标签分类、Markdown 渲染、公开分享链接。

## 原始需求

> 开发一个笔记管理系统，需要用户登录注册、笔记管理（增删改查、搜索、标签分类、Markdown渲染）、笔记公开分享链接

来源事件：`e000001` (owner_input), `e000003` (owner_confirmation)

## 业务模型

- User 1:N Note
- Note M:N Tag
- Note 1:N ShareLink
- User 1:N ShareLink
- ActivityLog（审计）

角色：user

## 产品流程

1. 注册/登录 → 演示账号 + 假数据
2. 笔记列表 → 搜索、标签筛选
3. 笔记编辑 → Markdown 实时预览
4. 分享 → 生成链接，未登录可查看
5. 异常：未登录拦截、分享过期/不存在

## 标准交付基线

- 前端：HTML/CSS/JS SPA（marked.js）
- 后端：FastAPI + SQLAlchemy + SQLite
- DB：schema + 初始化数据 + 迁移
- 测试：pytest 覆盖全部核心流程
- 文档：architecture.md / runbook.md / decisions/
- 环境：复用 ~/外部需求/.conda/codingagent
